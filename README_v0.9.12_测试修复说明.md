# v0.9.12 测试修复说明

本版本基于用户本次上传的工程继续修改，没有更换 Gradle Wrapper、AGP、依赖版本或工程构建环境。

## 本次重点检查

- XML 布局解析
- `@string` / `@color` / `@drawable` / `@layout` / `@id` 资源引用
- Java 中 `R.xxx` 资源引用
- 首页、阅读、运动作息、单词、日记自律、勋章境界、查看全部、详情页常见跳转
- 自定义弹窗替代系统弹窗的调用情况
- 查看全部页面长按删除和详情跳转
- 运动记录标题、体重趋势、单词最近记录的排序和展示逻辑

## 修复内容

1. **运动记录主标题修复**
   - 今日运动和历史运动列表只展示运动名称。
   - 距离、时长、热量放到副标题或详情页展示。

2. **运动记录兼容解析**
   - 新增 `ExerciseFormat` 工具类。
   - 统一解析旧数据中保存在 `content` 里的运动名称、时长和距离。
   - 避免多个页面重复写字符串切割逻辑。

3. **最近记录排序修复**
   - 运动历史按日期倒序展示。
   - 单词最近记录按日期倒序展示。
   - 体重曲线使用最近 7 条记录，并按时间正序绘制。
   - 当前体重取日期最新的一条，而不是数组最后一条。

4. **作息达标规则修复**
   - 入睡时间必须早于 23:00。
   - 起床时间必须不晚于 08:30。

5. **连续天数计算修复**
   - 如果今天还没打卡，但昨天之前连续打卡，当前连续天数不会直接显示为 0。
   - 更符合实际使用场景。

6. **阅读模块简化清理**
   - 书籍详情页不再暴露分页批注入口。
   - 阅读首页不再渲染最近分页批注。
   - `PageNote` 数据模型保留，用于兼容旧备份和旧数据。
   - 全书读后感入口保留在书籍详情页，并允许随时填写。

7. **全部记录和详情页稳定性修复**
   - `RecordListActivity` 和 `RecordDetailActivity` 增加空数据保护。
   - 运动详情页按运动名称、热量、时长、距离、日期分区展示。
   - 全部书籍点击仍跳转书籍详情页。

8. **统计口径调整**
   - 阅读笔记统计只统计全书读后感。
   - 不再把已经从 UI 移除的分页批注计入总笔记数量。

## 检查结果

- 打包时移除了 `local.properties` 和 `app/build/`，避免本机 SDK 路径和旧构建缓存影响导入。
